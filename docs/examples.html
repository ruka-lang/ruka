Hello World

The entry point of every Ruka program is a public const main function. It takes no arguments and returns () (unit). @println writes a string to standard output followed by a newline. 

const main = () do
    @println("Hello, world!")
end

Binary Tree

A recursive variant type where a tree is either a leaf holding an i32, or a branch holding two sub-trees. Methods on variants use pattern matching to dispatch on the active tag and recurse naturally into the structure. 

local Tree = variant {
    leaf:   i32,
    branch: record { left: Tree, right: Tree }
}

// Sum of all leaf values
const sum (self) = () do
    match self with
        .leaf(n)   => n
        .branch(b) => b.left.sum() + b.right.sum()
    end
end

// Height of the tree
const height (self) = () do
    match self with
        .leaf(_)   => 1
        .branch(b) do
            let l = b.left.height()
            let r = b.right.height()
            1 + if l > r do l else r end
        end
    end
end

const main = () do
    //       branch
    //      /      \
    //  branch     leaf(5)
    //  /    \
    // leaf(1) leaf(2)
    let tree = Tree.branch(.{
        left = Tree.branch(.{
            left  = Tree.leaf(1),
            right = Tree.leaf(2)
        }),
        right = Tree.leaf(5)
    })

    @println("sum:    ${tree.sum()}")     // 8
    @println("height: ${tree.height()}")  // 3
end
Note: Methods on variants follow the same receiver syntax as methods on records. The variant itself is the receiver type, and match inside the body handles each tag.

Shapes & Interfaces

Two unrelated record types, Circle and Rect, both satisfy the Shape interface because they each implement the required methods. No explicit declaration is needed — satisfaction is structural and implicit.

Because methods must be declared in the same scope as their type, the compiler can infer receiver types from field access. Circle is the only type in scope with a radius field, so self in those methods is unambiguously Circle — no annotation needed. Record literals follow the same logic: .{ radius = 5.0 } can only be a Circle.

Parameters typed with an interface use dynamic dispatch automatically — no extra keyword is required. Any value that satisfies the interface can be passed. 

local Shape = interface {
    area(self):      (): f64
    perimeter(self): (): f64
    label(self):     (): string
}

local Circle = record { radius: f64 }
local Rect   = record { width: f64, height: f64 }

// self inferred as Circle — only type in scope with `radius`
const area      (self) = () => 3.14159 * self.radius * self.radius
const perimeter (self) = () => 2.0 * 3.14159 * self.radius
const label     (self) = () => "Circle(r=${self.radius})"

// self inferred as Rect — only type with both `width` and `height`
const area      (self) = () => self.width * self.height
const perimeter (self) = () => 2.0 * (self.width + self.height)
const label     (self) = () => "Rect(${self.width}x${self.height})"

// Interface type implies dynamic dispatch — any Shape can be passed
local print_info = (s: Shape) do
    @println("${s.label()}")
    @println("  area:      ${s.area()}")
    @println("  perimeter: ${s.perimeter()}")
end

const main = () do
    let c = .{ radius = 5.0 }          // inferred as Circle
    let r = .{ width = 4.0, height = 6.0 }  // inferred as Rect

    print_info(c)
    // Circle(r=5.0)
    // area:      78.539...
    // perimeter: 31.415...

    print_info(r)
    // Rect(4.0x6.0)
    // area:      24.0
    // perimeter: 20.0
end

Generic Functions

When a parameter's type is type, the eva mode is inferred automatically — no keyword is needed. A function with such parameters is instantiated separately for each unique combination of compile-time type arguments, similar to monomorphisation or templates. Non-type compile-time parameters still require an explicit eva.

Compile-time functions can also return types, enabling parameterised type definitions. Pair below is a function that takes two types and produces a record type, which can then be used anywhere a type is expected. 

// Generic min
local min = (t, a: t, b: t) => if a < b => a else => b

// Compile-time type constructor — returns a parameterised record type
local Pair = (a, b) do
    record { first: a, second: b }
end

// Swap the two fields, producing a Pair with reversed type parameters
local swap = (a, b, p: Pair(a, b)) => Pair(b, a).{ first = p.second, second = p.first }

const main = () do
    // Each distinct type argument causes a new instantiation at compile time
    @println("min(3, 7)     = ${min(i32, 3, 7)}")
    @println("min(1.5, 2.0) = ${min(f64, 1.5, 2.0)}")

    let p = Pair(string, i32).{ first = "score", second = 42 }
    let q = swap(string, i32, p)
    @println("${q.first}: ${q.second}")  // 42: score
end
Note: Only values provably known at compile time may be passed to eva parameters. Passing a runtime value is a compile error.
