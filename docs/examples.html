Hello World

The entry point of every Ruka program is a public const main function. It takes no arguments and returns () (unit). @println writes a string to standard output followed by a newline. 

const main = () do
    @println("Hello, world!")
end

FizzBuzz

Print the numbers 1–100, replacing multiples of 3 with Fizz, multiples of 5 with Buzz, and multiples of both with FizzBuzz.

fizzbuzz uses an if/else chain as an expression — the whole chain evaluates to a single string that is returned implicitly as the last expression in the block. 

local fizzbuzz = (n: usize) do
    if n % 15 == 0 do "FizzBuzz"
    else if n % 3 == 0 do "Fizz"
    else if n % 5 == 0 do "Buzz"
    else do "${n}"
    end
end

const main = () do
    for i in 1..=100 do
        @println(fizzbuzz(i))
    end
end

Fibonacci

Two implementations of the Fibonacci sequence side by side. The recursive form is a direct translation of the mathematical definition. The iterative form runs in linear time and constant space.

The recursive version uses if/else as its return expression. The iterative version uses an early return for the base case and an implicit return of b at the end of the block. 

// Recursive — mirrors the mathematical definition; exponential time
local fib = (n: usize) do
    if n <= 1 do n
    else fib(n - 1) + fib(n - 2)
    end
end

const main = () do
    for i in 0..=10 do
        @println("fib(${i}) = ${fib(i)}")
    end
end

// fib(0) = 0
// fib(1) = 1
// fib(2) = 1
// fib(3) = 2
// ...
// fib(10) = 55

Binary Tree

A recursive variant type where a tree is either a leaf holding an i32, or a branch holding two sub-trees. Methods on variants use pattern matching to dispatch on the active tag and recurse naturally into the structure. 

local Tree = variant {
    leaf:   i32,
    branch: record { left: Tree, right: Tree }
}

// Sum of all leaf values
const sum (self) = () do
    match self with
        .leaf(n)   => n
        .branch(b) => b.left.sum() + b.right.sum()
    end
end

// Height of the tree
const height (self) = () do
    match self with
        .leaf(_)   => 1
        .branch(b) do
            let l = b.left.height()
            let r = b.right.height()
            1 + if l > r do l else r end
        end
    end
end

const main = () do
    //       branch
    //      /      \
    //  branch     leaf(5)
    //  /    \
    // leaf(1) leaf(2)
    let tree = Tree.branch(.{
        left = Tree.branch(.{
            left  = Tree.leaf(1),
            right = Tree.leaf(2)
        }),
        right = Tree.leaf(5)
    })

    @println("sum:    ${tree.sum()}")     // 8
    @println("height: ${tree.height()}")  // 3
end
Note: Methods on variants follow the same receiver syntax as methods on records. The variant itself is the receiver type, and match inside the body handles each tag.

Stack

A last-in first-out stack. The internal array and length counter are private (local fields), so callers can only interact through the public API. Static functions and methods are declared outside the type in the same scope, keeping the type definition focused on memory layout. Operations that may produce no value return option(T).

.? on an option(T) value force-unwraps the inner value, panicking at runtime if the option is .none. Here it is safe because the loop guard ensures the stack is non-empty before popping. 

local Stack = record {
    local data: [i32],
    local len:  usize
}

const new (Stack) = () => .{ data = .{}, len = 0 }

const push (mut self) = (value) do
    // append value to self.data
    self.len = self.len + 1
end

const pop (mut self) = () do
    if self.len == 0 do
        .none
    else do
        self.len = self.len - 1
        .some(self.data[self.len])
    end
end

const peek (self) = () do
    if self.len == 0 do .none
    else .some(self.data[self.len - 1])
    end
end

const is_empty (self) = () => self.len == 0

const size (self) = () => self.len

const main = () do
    let s = Stack.new()

    s.push(10)
    s.push(20)
    s.push(30)

    @println("size: ${s.size()}")      // 3
    @println("peek: ${s.peek().?}")    // 30

    while not s.is_empty() do
        @println("pop: ${s.pop().?}")
    end
    // pop: 30
    // pop: 20
    // pop: 10
end

Shapes & Interfaces

Two unrelated record types, Circle and Rect, both satisfy the Shape interface because they each implement the required methods. No explicit declaration is needed — satisfaction is structural and implicit.

Because methods must be declared in the same scope as their type, the compiler can infer receiver types from field access. Circle is the only type in scope with a radius field, so self in those methods is unambiguously Circle — no annotation needed. Record literals follow the same logic: .{ radius = 5.0 } can only be a Circle.

Parameters typed with an interface use dynamic dispatch automatically — no extra keyword is required. Any value that satisfies the interface can be passed. 

const Shape = interface {
    area(self):      (): f64
    perimeter(self): (): f64
    label(self):     (): string
}

local Circle = record { radius: f64 }
local Rect   = record { width: f64, height: f64 }

// self inferred as Circle — only type in scope with `radius`
const area      (self) = () => 3.14159 * self.radius * self.radius
const perimeter (self) = () => 2.0 * 3.14159 * self.radius
const label     (self) = () => "Circle(r=${self.radius})"

// self inferred as Rect — only type with both `width` and `height`
const area      (self) = () => self.width * self.height
const perimeter (self) = () => 2.0 * (self.width + self.height)
const label     (self) = () => "Rect(${self.width}x${self.height})"

// Interface type implies dynamic dispatch — any Shape can be passed
const print_info = (s: Shape) do
    @println("${s.label()}")
    @println("  area:      ${s.area()}")
    @println("  perimeter: ${s.perimeter()}")
end

const main = () do
    let c = .{ radius = 5.0 }          // inferred as Circle
    let r = .{ width = 4.0, height = 6.0 }  // inferred as Rect

    print_info(c)
    // Circle(r=5.0)
    // area:      78.539...
    // perimeter: 31.415...

    print_info(r)
    // Rect(4.0x6.0)
    // area:      24.0
    // perimeter: 20.0
end

Generic Functions

When a parameter's type is type, the eva mode is inferred automatically — no keyword is needed. A function with such parameters is instantiated separately for each unique combination of compile-time type arguments, similar to monomorphisation or templates. Non-type compile-time parameters still require an explicit eva.

Compile-time functions can also return types, enabling parameterised type definitions. Pair below is a function that takes two types and produces a record type, which can then be used anywhere a type is expected. 

// Generic min
const min = (t, a: t, b: t) => if a < b do a else b end

// Compile-time type constructor — returns a parameterised record type
const Pair = (a, b) do
    record { first: a, second: b }
end

// Swap the two fields, producing a Pair with reversed type parameters
const swap = (a, b, p: Pair(a, b)) => Pair(b, a).{ first = p.second, second = p.first }

const main = () do
    // Each distinct type argument causes a new instantiation at compile time
    @println("min(3, 7)     = ${min(i32, 3, 7)}")
    @println("min(1.5, 2.0) = ${min(f64, 1.5, 2.0)}")

    let p = Pair(string, i32).{ first = "score", second = 42 }
    let q = swap(string, i32, p)
    @println("${q.first}: ${q.second}")  // 42: score
end
Note: Only values provably known at compile time may be passed to eva parameters. Passing a runtime value is a compile error.

Pipelines

The |> operator passes the left-hand value as the first argument of the right-hand call. This lets you write a chain of operations in the order they are applied — top to bottom, left to right — rather than inside out.

Because |> always inserts the left-hand value as the first argument, it works uniformly with free functions, static functions, methods, and built-ins. The example below passes a Dog value through a series of transformation methods and then hands the final string to @println. 

local Dog = record { name: string, breed: string, age: isize }

const new (Dog) = (name, breed) => Dog.{ name, breed, age = 0 }

// Each method returns a new Dog so the value can flow through a pipeline
const with_name (self) = (name) => Dog.{ name, breed = self.breed, age = self.age }

const with_age (self) = (age) => Dog.{ name = self.name, breed = self.breed, age }

const bio (self) = () => "${self.name}, ${self.breed}, age ${self.age}"

const main = () do
    // Each |> feeds the result of the left side into the first parameter
    // of the right side — Dog flows through until @println receives a string.
    Dog.new("puppy", "Labrador")
        |> with_name("Rex")
        |> with_age(3)
        |> bio()
        |> @println()
    // Rex, Labrador, age 3
end

Named Parameters

A ~ prefix on a parameter makes it named — callers must supply it by label using ~label=value syntax. Named parameters can be passed in any order and may trail after the closing parenthesis, which is especially useful for passing closures to higher-order functions.

The example below defines map, filter, and reduce with a named closure parameter ~f (or ~pred) placed last. At the call site the closure appears as a trailing block, making the code read like a language built-in. 

// Higher-order functions with named closure parameters
const map    = (t, u, arr: [t], ~f: (t): u): [u] do ... end
const filter = (t, arr: [t], ~pred: (t): bool): [t] do ... end
const reduce = (t, u, arr: [t], init: u, ~f: (u, t): u): u do ... end

let nums = .{1, 2, 3, 4, 5}

// Single-line trailing block
let doubled = map(nums) ~f=(x) => x * 2
// .{2, 4, 6, 8, 10}

let evens = filter(nums) ~pred=(x) => x % 2 == 0
// .{2, 4}

let total = reduce(nums, 0) ~f=(acc, x) => acc + x
// 15

// Multi-line trailing block — scales naturally to complex bodies
let results = map(nums) ~f=(x) do
    let squared = x * x
    squared - 1
end
// .{0, 3, 8, 15, 24}

Named Parameters for Clarity

Named parameters are also useful when a function has multiple parameters of the same type and the positional order would be easy to confuse. 

const replace = (s: string, ~from: string, ~to: string): string do ... end

// The label makes the intent clear at the call site
let result = replace("hello world", ~from="world", ~to="Ruka")
// "hello Ruka"

// Labels can be supplied in any order
let result2 = replace("hello world", ~to="Ruka", ~from="world")
// "hello Ruka"

// Shorthand — variable name matches the label
let from = "world"
let to   = "Ruka"
let result3 = replace("hello world", ~from, ~to)
