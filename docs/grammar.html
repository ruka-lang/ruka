Notation

This document specifies the Ruka language grammar using a variant of Extended Backus–Naur Form (EBNF). Each production rule has the form: 

rule-name  ::=  production

The following meta-syntax is used throughout:

A BConcatenation — A followed by B.
A | BAlternation — A or B.
A?Option — zero or one occurrence of A.
A*Repetition — zero or more occurrences of A.
A+One or more occurrences of A.
( A )Grouping — treats A as a single unit.
"terminal"A literal character sequence.
[a-z]A character class — any character in the given range.
<description>An informal prose description of a terminal set.
-- commentAn explanatory note; not part of the production.

Lexical rules (tokens) are written in lowercase-with-hyphens and describe raw text. Syntactic rules also use lowercase-with-hyphens and describe token sequences. The two levels are separated into distinct sections below. Whitespace and comments between tokens are always permitted and are omitted from syntactic rules for clarity.

Lexical Grammar

The lexical grammar describes how raw source text is divided into tokens. The scanner reads left-to-right and always produces the longest possible token at each position (maximal munch).

Source Encoding

Ruka source files are encoded in UTF-8. The term char below means any Unicode scalar value.

Whitespace & Comments

Whitespace and comments are discarded by the scanner and do not appear in the syntactic grammar. Block comments are not supported. 

whitespace    ::=  ( " " | "\t" | "\r" | "\n" )+

comment       ::=  "//" <any char except "\n">* "\n"?

Identifiers

An identifier begins with a letter or underscore and is followed by zero or more letters, digits, or underscores. Any identifier that matches a keyword or mode keyword is reserved and cannot be used as a user-defined name. 

letter        ::=  [a-zA-Z_]
digit         ::=  [0-9]

identifier    ::=  letter ( letter | digit )*
                   -- must not be a keyword or mode keyword

Keywords & Modes

The following identifiers are reserved as keywords:

and break const continue defer do else end false for if in interface let local match not or record return self test true variant while with

The following identifiers are reserved as mode keywords. They are only valid immediately before a parameter identifier or a binding identifier; in all other positions they are parse errors. 

mode          ::=  "mut"    -- mutable reference; changes are visible to the caller
               |   "mov"    -- ownership transfer; caller cannot use the value after the call
               |   "stc"    -- stack-allocated copy; value cannot escape the function scope
               |   "eva"    -- compile-time value; must be known at compile time
self is a reserved keyword that refers to the method receiver instance. It may only appear in the receiver position of a binding declaration or as a parameter inside the function body of a method. with is a keyword used exclusively as part of the match expression syntax.

Literals

Integer Literals

decimal-digit ::=  [0-9]
hex-digit     ::=  [0-9a-fA-F]
binary-digit  ::=  "0" | "1"

decimal-lit   ::=  decimal-digit+
hex-lit       ::=  "0x" hex-digit+
binary-lit    ::=  "0b" binary-digit+

integer-lit   ::=  decimal-lit | hex-lit | binary-lit
                   -- a leading minus sign is parsed as a unary operator, not part of the literal
                   -- default type: isize; overridden by type context

Float Literals

float-lit     ::=  decimal-digit+ "." decimal-digit+
                   -- leading minus is a unary operator
                   -- default type: float; overridden by type context

Boolean Literals

bool-lit      ::=  "true" | "false"

String Literals

String literals are delimited by double quotes. Expressions may be embedded with ${…}; the interpolated expression may be any Ruka expression whose value is convertible to string. 

str-escape    ::=  "\\" ( "n" | "t" | "r" | "\\" | '"' )
str-interp    ::=  "${" expr "}"
str-content   ::=  ( <any char except '"', '\\', or '$'> | str-escape | str-interp )*
string-lit    ::=  '"' str-content '"'

Multiline String Literals

Multiline strings begin with |" followed by a newline. Each content line must begin with |, which is stripped from the resulting string value. The literal ends with |". 

ml-content    ::=  <any char except "\n">
ml-line       ::=  "|" ml-content* "\n"
multiline-lit ::=  '|"' "\n" ml-line* '|"'

Syntactic Grammar

The syntactic grammar is defined over token sequences. Whitespace and comments between any two tokens are implicitly permitted and ignored.

Program

A Ruka source file is a flat sequence of declarations. Every file is implicitly a compile-time record: const declarations become its public fields and local declarations are private fields inaccessible to importers. @import returns this record value; there is no separate module concept. 

program       ::=  declaration*

Declarations

All declarations share a single binding form. The binding keyword determines mutability and visibility; there is no separate fn, type, or mod keyword. 

declaration   ::=  binding

binding       ::=  binding-kw binding-lhs "=" expr
               |   binding-kw binding-lhs ":" type "=" expr

binding-kw    ::=  "let"     -- mutable; scoped to the enclosing block
               |   "const"   -- immutable; public (exported from its module)
               |   "local"   -- immutable; private (not exported)
               |   "test"    -- test function; compiled only in debug and test builds

binding-lhs   ::=  mode? identifier                  -- simple value binding
               |   mode? identifier receiver          -- function or method binding
               |   destructure-pattern                -- destructuring binding

receiver      ::=  "(" type-receiver ")"
type-receiver ::=  identifier                        -- static function: associated type name
               |   mode? "self"                      -- method: instance receiver

destructure-pattern
              ::=  ".{" identifier ( "," identifier )* "}"
                   -- destructures any record value, including imported files
                   -- e.g.  local .{ x, y } = point
                   -- e.g.  local .{ sqrt, pow } = @import("math")
Uniform declarations. Functions, types, interfaces, and modules are all values stored in ordinary bindings. A receiver in the binding left-hand side associates the value with a named type as either a static function (type-name receiver) or a method (self receiver). The receiver appears between the binding name and the = sign.
Test bindings. A test binding declares a zero-argument function that is only compiled in debug and test builds; it is elided entirely in optimised builds. The value must be a function expression. Test bindings have no visibility — they are never exported — and a receiver clause is not permitted. The name is an ordinary identifier used by the test runner to identify the test.

Types

Type expressions appear after : in parameter and binding annotations, after -> in return-type annotations, and as values passed to eva-mode parameters. 

type          ::=  "()"                              -- unit type
               |   primitive-type
               |   array-type
               |   tuple-type
               |   hashmap-type
               |   option-type
               |   result-type
               |   function-type
               |   record-type
               |   variant-type
               |   interface-type
               |   identifier                        -- named type (user-defined or type parameter)

-- ── Primitive types ───────────────────────────────

primitive-type
              ::=  "isize"                           -- signed integer, platform-native width (default)
               |   "usize"                           -- unsigned integer, platform-native width
               |   "i8"   | "i16"  | "i32"  | "i64"  | "i128"
               |   "u8"   | "u16"  | "u32"  | "u64"  | "u128"
               |   "float"                           -- floating-point, platform-native width (default)
               |   "f32"  | "f64"
               |   "string"                          -- UTF-8 string
               |   "bool"                            -- boolean
               |   "type"                            -- the type of types; used in compile-time contexts

-- ── Collection types ──────────────────────────────

array-type    ::=  "[" type "]"                      -- homogeneous array: [i32]
tuple-type    ::=  "[" type ( "," type )+ "]"        -- fixed-length heterogeneous tuple: [i32, string]
hashmap-type  ::=  "[" type "=>" type "]"            -- key-value map: [string => usize]

-- ── Generic built-in types ────────────────────────

option-type   ::=  "option" "(" type ")"             -- value that may be absent
result-type   ::=  "result" "(" type "," type ")"    -- value that may be an error

-- ── Function type ─────────────────────────────────

function-type ::=  "(" type-list? ")" "->" type
type-list     ::=  type ( "," type )*

-- ── User-defined types ────────────────────────────

record-type   ::=  "record" "{" field-list? "}"
field-list    ::=  field ( "," field )* ","?
field         ::=  "local"? identifier ":" type      -- "local" marks a private field

variant-type  ::=  "variant" "{" tag-list? "}"
tag-list      ::=  tag ( "," tag )* ","?
tag           ::=  identifier ( ":" type )?           -- payload type is optional; absent means unit

interface-type
              ::=  "interface" "{" method-sig* "}"
method-sig    ::=  identifier "(" mode? "self" ")" ":" function-type
                   -- each line declares one required method signature
Result-location semantics. The collection literal .{…} may produce an array, tuple, or hashmap depending on the type expected by the surrounding context. When no context is available the literal defaults to a tuple. An explicit type annotation on the binding is the most common way to request an array or hashmap.

Expressions

Ruka is expression-based: blocks, conditionals, match expressions, and function bodies all evaluate to a value. The following layered grammar encodes operator precedence structurally, from lowest-binding (outermost rule) to highest-binding (innermost rule). 

expr          ::=  assign-expr

assign-expr   ::=  pipeline-expr ( "=" assign-expr )?
                   -- right-associative; left-hand side must be a mutable place expression

pipeline-expr ::=  or-expr ( "|>" or-expr )*
                   -- left-associative; inserts the left value as the first argument of the right call

or-expr       ::=  and-expr ( "or" and-expr )*

and-expr      ::=  not-expr ( "and" not-expr )*

not-expr      ::=  "not" not-expr
               |   compare-expr

compare-expr  ::=  range-expr ( compare-op range-expr )?
compare-op    ::=  "==" | "!=" | "<" | "<=" | ">" | ">="
                   -- non-associative; chaining two comparisons requires explicit parentheses

range-expr    ::=  add-expr ( ( ".." | "..=" ) add-expr )?
                   -- ".."  — exclusive: lower <= i < upper
                   -- "..=" — inclusive: lower <= i <= upper

add-expr      ::=  mul-expr ( ( "+" | "-" ) mul-expr )*

mul-expr      ::=  unary-expr ( ( "*" | "/" | "%" ) unary-expr )*

unary-expr    ::=  "-" unary-expr
               |   postfix-expr

Operator Precedence

The table below summarises all operators from lowest to highest precedence.

Level Operator(s) Associativity Description
1 (lowest)=RightAssignment
2|>LeftPipeline (reverse application)
3orLeftLogical OR
4andLeftLogical AND
5notPrefixLogical NOT
6==   !=   <   <=   >   >=NoneComparison
7..   ..=NoneRange construction
8+   -LeftAdditive arithmetic
9*   /   %LeftMultiplicative arithmetic

11 (highest).   []   ()   .?()   .!()LeftField access, index, call, unwrap

Postfix Expressions

Postfix operations bind tighter than any prefix or infix operator and are left-associative, allowing arbitrary chaining. 

postfix-expr  ::=  primary postfix-op*

postfix-op    ::=  "." identifier                         -- field access
               |   "." identifier "(" arg-list? ")"       -- method call
               |   ".?()"                                 -- option force-unwrap; panics if .none
               |   ".!()"                                 -- result force-unwrap; panics if .err
               |   "[" expr "]"                           -- index (array, tuple, hashmap, range slice)
               |   "(" arg-list? ")" trailing-arg*        -- function call

arg-list      ::=  arg ( "," arg )*
arg           ::=  named-arg | expr

named-arg     ::=  "~" identifier ( "=" expr )?
                   -- "~label=value" passes a labelled argument
                   -- "~label" shorthand: variable in scope shares the label name

trailing-arg  ::=  "~" identifier "=" function-expr
                   -- a named closure argument passed after the closing parenthesis
                   -- e.g.  map(nums) ~f=(x) => x * 2

Primary Expressions

primary       ::=  literal-expr
               |   identifier
               |   builtin-call
               |   block-expr
               |   if-expr
               |   match-expr
               |   while-expr
               |   for-expr
               |   return-expr
               |   break-expr
               |   continue-expr
               |   function-expr
               |   collection-lit
               |   record-lit
               |   variant-ctor
               |   "(" expr ")"                           -- parenthesised expression

-- ── Literal expressions ───────────────────────────

literal-expr  ::=  integer-lit
               |   float-lit
               |   bool-lit
               |   string-lit
               |   multiline-lit

-- ── Block expression ──────────────────────────────
-- A sequence of statements; evaluates to its last expression.

block-expr    ::=  "do" stmt* "end"
stmt          ::=  declaration | defer-stmt | expr

-- ── Collection literals ───────────────────────────
-- Array, tuple, or hashmap — the type is resolved from surrounding context.

collection-lit
              ::=  ".{" "}"                               -- empty (defaults to empty tuple)
               |   ".{" hashmap-items "}"                 -- hashmap (keys and values separated by "=>")
               |   ".{" expr-items "}"                    -- array or tuple (resolved by context)

hashmap-items ::=  hashmap-pair ( "," hashmap-pair )* ","?
hashmap-pair  ::=  expr "=>" expr

expr-items    ::=  expr ( "," expr )* ","?

-- ── Record literals ───────────────────────────────
-- Constructs a value of a record type.

record-lit    ::=  ( identifier "." )? ".{" field-inits? "}"
                   -- optional "TypeName." prefix forces the type; omitted when the type is inferrable

field-inits   ::=  field-init ( "," field-init )* ","?
field-init    ::=  identifier "=" expr                    -- explicit:   name = value
               |   identifier                             -- shorthand:  variable name matches field name

-- ── Variant constructors ──────────────────────────
-- Constructs a value of a variant type.

variant-ctor  ::=  "." identifier                         -- unqualified, no payload: .tagName
               |   "." identifier "(" expr ")"            -- unqualified, with payload: .tagName(expr)
               |   identifier "." identifier              -- qualified, no payload:   Type.tagName
               |   identifier "." identifier "(" expr ")" -- qualified, with payload: Type.tagName(expr)

Function Expressions

Functions are anonymous values. A function expression is a parameter list, an optional return-type annotation, and a body. The body is either a single expression (introduced with =>, requiring no closing end) or a do…end block. 

function-expr ::=  "(" param-list? ")" return-annot? function-body

return-annot  ::=  "->" type

function-body ::=  "=>" expr        -- single-expression body; no "end" required
               |   block-expr       -- multi-statement body

param-list    ::=  param ( "," param )*
param         ::=  mode? "~"? identifier type-annot?     -- positional or named parameter
               |   mode? "self"                          -- method receiver (keyword only)

type-annot    ::=  ":" type
Receiver and function expression. When a binding declaration carries a receiver clause, the param-list describes only the explicit parameters — the receiver itself is declared by the binding, not by the function expression. See Declarations.

Named Parameters

Prepending ~ to a parameter name makes it a named parameter. Named parameters are always passed by label at the call site and may appear in any order. A named parameter may also be passed as a trailing argument outside the closing parenthesis, which is useful for higher-order functions that accept a closure. 

-- Declaration (in param-list)
--   mode? "~" identifier type-annot?
--   e.g.  ~name: string,  ~f: (isize) -> isize

-- Call site (in arg-list)
--   "~" identifier "=" expr     standard form:  ~name="Ruka"
--   "~" identifier              shorthand form: ~name  (variable in scope has the same name)

-- Trailing form (after the closing parenthesis)
--   "~" identifier "=" function-expr
--   e.g.  map(nums) ~f=(x) => x * 2
--   e.g.  reduce(nums, 0) ~f=(acc, x) do
--             acc + x
--         end

Control Flow

All control flow constructs are expressions and produce a value. When used purely for side effects the result is the unit type ().

If Expression

if-expr       ::=  "if" expr if-branch
                   ( "else" "if" expr if-branch )*
                   else-branch?
                   "end"
                   -- a single "end" closes the entire if chain

if-branch     ::=  "do" stmt*       -- block body; opened with "do", closed by the chain's "end"

else-branch   ::=  "else" "do" stmt*    -- else with a block body
               |   "else" expr          -- bare else expression without "do"
                   -- e.g.  else "C"  in  if score >= 90 do "A" else if score >= 80 do "B" else "C" end

While Expression

while-expr    ::=  "while" expr "do" stmt* "end"

For Expression

for-expr      ::=  "for" for-pattern "in" expr "do" stmt* "end"

for-pattern   ::=  identifier                             -- simple binding
               |   ".{" identifier ( "," identifier )* "}"
                   -- tuple destructuring in the loop variable
                   -- e.g.  for .{k, v} in pairs do ... end

Return

return-expr   ::=  "return" expr?
                   -- "return" with no expression returns the unit value ()

Break and Continue

break and continue are only valid inside a while or for loop body. Both produce the unit value. 

break-expr    ::=  "break"
                   -- immediately exits the innermost enclosing loop

continue-expr ::=  "continue"
                   -- skips the remainder of the current iteration and begins the next

Defer

A defer statement schedules an expression to execute at the end of the enclosing do…end block, regardless of how control exits that block. Multiple defers in the same block execute in LIFO order (last deferred, first to run). 

defer-stmt    ::=  "defer" expr
                   -- expr is evaluated when the enclosing block exits
                   -- expr is typically a function call or a do...end block
                   -- defers in the same block run LIFO: last defer statement runs first

Patterns

Patterns appear in match arms. Arms are tested top-to-bottom; the first matching arm is taken. The compiler requires exhaustive coverage: every possible value must be handled by some arm. 

match-expr    ::=  "match" expr "with" arm+ "end"

arm           ::=  pattern arm-body

arm-body      ::=  "=>" expr              -- single-expression arm
               |   "do" stmt* "end"       -- block arm

pattern       ::=  variant-pattern        -- matches a variant tag and optionally binds its payload
               |   literal-expr           -- matches an exact value (integer, float, bool, string)
               |   guard-pattern          -- a boolean expression used as a conditional arm
               |   "else"                 -- catch-all; must be the last arm

variant-pattern
              ::=  "." identifier
                   -- matches a tag with no payload:       .tagName
               |   "." identifier "(" payload-pattern ")"
                   -- matches a tag with a payload and binds it

payload-pattern
              ::=  identifier
                   -- binds the entire payload to a single name:   .circle(r)
               |   ".{" identifier ( "," identifier )* "}"
                   -- destructures a tuple payload:                .rectangle(.{w, h})

guard-pattern ::=  expr
                   -- any boolean expression; may reference the matched value by its binding name
                   -- e.g.  n < 0   in   match n with  n < 0 => "negative" ... end
Option and result patterns. option(T) and result(T, E) are built-in variant types and follow the same variant-pattern syntax: .some(name), .none, .ok(name), .err(name).

Built-in Calls

Built-in functions are prefixed with @ to distinguish them from user-defined names. They are available in every scope without an import and cannot be redefined by user code. 

builtin-call  ::=  "@" identifier "(" arg-list? ")"

-- Standard built-ins
-- @print(string)              print to stdout without a trailing newline
-- @println(string)            print to stdout with a trailing newline
-- @import(string-lit)         evaluate at compile time; returns the file as a compile-time record value
-- @typeOf(expr)               return the compile-time type of an expression as a "type" value
-- @abs(number)                absolute value
-- @sqrt(number)               square root
-- @len(collection)            length of an array, tuple, or string
-- @assertEq(a, b)             assert equality; panics at runtime on failure
-- @compileError(string-lit)   emit a compile-time error with the given message
