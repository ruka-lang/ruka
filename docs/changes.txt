- Destructuring used as boolean `if .some(x) = var do...end` `while .some(x) = var do...end` and can iterate only handling matching patterns `for .some(x) in arr do...end`.
- Stack bindings can be moved as well, will not be GC'd or copied, passed-by-ptr
- std library will include allocators that will return references with program lifetimes that must be manually freed
- Highlight potential compiler extensiablilty for plugins allowing domain specific features to be added
- `_` useable as identifier to ignore something
- Char literals which are infered as u8
- Define more operators like bitwise(&, |), exponent (^), etc
- Named parameters with option type infer .none when not provided and will wrap value passed with .some()
- local and let are for runtime values. const can only be used for defining static values
- Return annotated as a result or option will wrap returned value based on inferrence. ie if result(f64, string) returning a string will become .err(string). for option(usize) returning unit () will become .none and a usize will become .some(usize)
- Only one trailing argument allowed
- Any value attachable as a static not just functions
- Files read as empty records w/ each declaration stored as a static field. Empty records uninstanciable
- @now() builtin that returns current time as a timestamp
- @size() builtin for calculating bytes of type
- Arrays, tuples, and maps can be created with `[type].{...}`, `[type, type].{...}`, `[type => type].{...}` as well (idiomatic way for arrays and maps). This mirrors how records and variants are initialized

- Single line blocks apply to anything that accepts a block, including branches and loops
- Single line branching
if cond => this else => that
- Single line loops
while cond => code
for 0..10 => code
for i in 0..10 => code

- Array comprehensions for initializing array values
[f64].{ for 0..3 => random.float(f64, -1, 1) }
- Map comprehensions
let arr = [f64].{ 1.0, 2.0, 3.0, 4.0 }
[string => float].{ for e in arr => .{"number ${e}" => e} }
