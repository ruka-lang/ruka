What is Ruka?

Ruka is a general-purpose compiled programming language designed around clarity and expressiveness.

Design Principles

  • Uniform declarations. Functions, types, records, and variants (tagged unions) are all values stored in bindings, files are imported as records as well. There is one syntax for naming things.
  • Expression-based. Blocks, conditionals, match expressions, and function bodies all evaluate to a value. The last expression in a block is its result.
  • Garbage collected with opt-in lifetime control. The GC handles memory by default but the user can decide when to control lifetimes or stack/heap allocate.
  • Compile-time evaluation and interpretation allowing generic programming via static dispatch.
  • Behaviours not inheritance. Ruka has no inheritance. Shared behaviour is expressed through behaviours, giving the function an opaque representation of the argument seeing only a subset of it's behaviour.
  • Methods live outside types. A type definition describes only memory layout, the instance fields. Methods and static fields are declared separately in the same scope, keeping type definitions minimal and readable.

Core Features

Interpreted and Compilation

Ruka ships both an interpreter and a compiler. The compiler uses the interpreter for metaprogramming and the user can use the interpreter for debugging and rapid prototyping.

Strong Static Typing

Aggressive type inference — annotations are only written where the type cannot be uniquely determined. Ambiguity is a compile error, never a silent fallback.

Garbage Collection

Memory is managed automatically. Parameter modes allow opt-in lifetime control and stack/heap allocations.

Expression-Based

Blocks, if/else, match, and destructuring all return values. No statement/expression split.

Pattern Matching

Exhaustive match over variants, literals, and guard conditions. Most types can be destructured and those expressions return booleans.

Built-in Testing

Test functions are first-class language constructs declared with test. They compile only in debug and test builds — zero overhead and zero code in production binaries.

Behaviours

Structural typing via interfaces. Any type matching the method signatures satisfies an interface implicitly. Parameters typed with an interface cause that function to be compiled for the argument type passed in.

Compile-Time Evaluation

Ruka code can be interpreted at compile time to generate types, instantiate generics, and generate constants that require more complex calculations.

Built-in Collections

Arrays, tuples, and hashmaps are built into the type system with literal syntax.

Development

Philosophy

  • Fully functional interpreter. The language should be runnable without compilation, enabling rapid iteration and scripting use cases.
  • Custom compiler backend. Ruka targets its own backend rather than relying on LLVM or similar, keeping the toolchain self-contained and tightly integrated with the language.
  • Self-hosted tooling. The compiler, interpreter, and tools are eventually written in Ruka itself.
  • A singular tool. Project management, package management, testing, and editor integration are all provided under one unified ruka command — no fragmented ecosystem.

Goals

Short-Term

  • Basic interpreter
  • Basic project management
  • Basic editor support (tree-sitter grammar and basic LSP)
  • Basic compiler targeting AArch64

Mid-Term

  • Self-hosting of tooling
  • Expanded interpreter and compiler with basic optimizations
  • Project management with full testing support
  • More comprehensive editor support
  • Basic package management (using GitHub repositories)
  • AMD64 and WASM compilation targets

Long-Term

  • Additional compilation targets
  • Fully featured interpreter and compiler with advanced optimizations
  • Compiler and interpreter plugin system

Future Plans

The current focus is on the core language. The following extensions are planned for the future. Potentially allowing custom compiler/interpreter extensions to be made for domain specific solutions.

Ruka.web

First-class JSX-like syntax for HTML and CSS directly in Ruka source, enabling server-side rendering without a template engine. Components would be ordinary Ruka functions.

Ruka.hdl

A hardware description language using Ruka syntax, targeting FPGA development on SoC boards such as the Xilinx Zynq family. Hardware modules expose getter and setter interfaces, allowing the same language to describe both the software and the hardware it runs on.
